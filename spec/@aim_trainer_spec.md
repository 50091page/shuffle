# 에임 훈련장 (Aim Trainer) 미니게임 구현 명세서

## 1. 개요
현재 개발 중인 '스트리머 팀 짜기 웹사이트' 내에 미니게임으로 추가될 에임 훈련(Aim Trainer) 페이지입니다. 전문적인 에임 훈련 프로그램 수준의 렌더링 성능과 시각적 디테일, 타격감을 구현하는 것이 핵심 목표입니다.

## 2. 기술 스택
- **렌더링:** `@pixi/react` (DOM 대신 Canvas/WebGL 렌더링으로 프레임 방어)
- **상태 관리:** `zustand` (리렌더링 최소화 및 게임 루프 최적화)

---

## 3. 구현 요구사항 및 체크리스트

### Phase 1: 진입점 및 라우팅 연결
- [x] 메인 화면의 '추가기능 준비중...' 카드 컴포넌트 식별
- [x] 해당 카드의 텍스트와 디자인을 '에임 훈련장 (Aim Trainer)'으로 변경
- [x] 카드를 클릭 시 `<AimTrainer />` 페이지/컴포넌트로 이동하도록 라우팅 연결
- [x] `<AimTrainer />` 화면 내에 메인 화면으로 돌아갈 수 있는 '뒤로 가기(메인으로)' 버튼 추가

### Phase 2: 상태 관리 (`zustand` 스토어)
- [x] `useAimStore` 구성 (상태: `score`, `misses`, `timeLeft`, `gameState`, `targetDuration`)
- [x] `gameState`는 `idle`(대기), `playing`(진행 중), `finished`(종료)로 관리
- [x] **난이도 점진적 상승 로직 (핵심):**
  - `targetDuration`(표적 유지 시간)의 초기값은 `1000ms`로 설정
  - 표적 적중(Hit) 시 `score` +1 증가 및 `targetDuration`을 `30ms`씩 감소
  - 단, `targetDuration`은 하한선 `Math.max(400, prev - 30)`를 적용하여 400ms(발로란트 어려움 난이도 수준) 이하로 내려가지 않도록 제한

### Phase 3: UI/UX 디테일
- [x] **커스텀 크로스헤어:** 캔버스(게임 영역) hover 시 시스템 기본 마우스 커서를 숨기고(`cursor: none`), 마우스 좌표를 따라다니는 커스텀 십자선 그래픽 렌더링
- [x] **오버레이 UI:** 캔버스 내부 상단 또는 중앙 영역에 `현재 점수`, `Miss 횟수`, `남은 시간`을 세련된 텍스트 오버레이로 표시 (게임 배경과 대비되는 모던한 폰트/디자인)

### Phase 4: 표적(Target) 로직 및 애니메이션
- [x] 게임 시작 시 지정된 캔버스 영역 내 랜덤한 좌표에 원형 표적 생성 루프 구현
- [x] **스폰(Spawn) 애니메이션:** 표적이 생성될 때 크기(Scale)가 0에서 1로 튀어 오르듯(Pop-up) 부드럽게 나타나게 구현
- [x] **타격(Hit) 피드백:** 표적 클릭 성공 시 즉시 삭제하지 않고, 스케일이 빠르게 작아지거나 색상이 번쩍인 후 사라지는 타격 애니메이션 적용
- [x] **미스(Miss) 처리:** 현재의 `targetDuration` 시간 내에 표적을 클릭하지 못하면 `misses` 카운트를 1 증가시키고, 표적이 투명해지며(Fade-out) 자연스럽게 사라진 후 다음 표적이 생성되도록 구현
